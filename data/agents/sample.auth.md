# Sub-Agent: Auth

## Role
- 인증/인가 흐름(로그인, 세션/토큰, 권한 체크)의 책임을 가진다.
- UI/도메인 로직과 분리되어야 하며, 인증 상태는 최소한의 형태로 노출한다.

## Design Decisions
- 인증 상태는 서버에서 검증한다(클라이언트 신뢰 금지).
- 민감한 키/토큰은 클라이언트 번들에 포함하지 않는다.
- 라우트 보호는 서버 라우트/미들웨어 단에서 우선 처리한다.

## Constraints
- API Key(예: OpenAI)는 반드시 서버에서만 사용한다.
- 인증 관련 변경은 기존 플로우/결정을 깨지 않도록 최소 변경으로 적용한다.

## Prompt Blueprint (옵션)
- 변경 요청이 Auth 관련이면, 먼저 "현재 인증 방식/저장 위치/보호 대상 라우트"를 확인하고 필요한 최소 변경을 제안한다.
