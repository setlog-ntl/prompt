# Project Core

## Purpose
- 바이브코딩 시 프로젝트의 "판단 맥락"을 보존하고, 사용자의 자유 입력을 즉시 실행 가능한 프롬프트로 변환한다.

## Philosophy / Principles
- 기억은 "설계 결정(Design Decisions)"만 저장한다.
- 새로운 설계 결정을 발명하지 않는다.
- 모호하면 질문을 먼저 한다.
- 변경 단위(프로젝트/서브에이전트)는 사용자가 선택한다.

## Long-term Direction
- Markdown 기반의 문서/결정 저장을 중심으로 확장한다.
- (옵션) Responses API store/previous_response_id로 상태 유지 패턴을 지원할 수 있다.
