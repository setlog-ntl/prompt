# PRD — VibePrompt OS Web (MVP)

## 1. Problem
바이브코딩에서 프롬프트가 휘발되고, 프로젝트/모듈 맥락이 일관되지 않으며, PM 관점의 체크가 누락된다.

## 2. Target User (Primary)
- 1인 바이브코더(개발자/빌더/PM)
- 프로젝트를 빠르게 만들되, "판단"은 누적하고 싶다.

## 3. Goal
- 자유 입력을 기반으로 실행용 프롬프트를 즉시 생성한다.
- 프로젝트 core + 선택된 sub-agent 맥락을 결합한다.
- 설계 결정만 기억하며, 프롬프트는 일부 재구성(정리/중복 제거)까지 허용한다.

## 4. Non-Goal (MVP 제외)
- 수익화/공유/협업
- 자동 코드 실행
- 장황한 히스토리 로그

## 5. UX Flow (MVP)
1) 변경 단위 선택
- Project / Sub-Agent(복수 선택 가능, 기본 1개)

2) 입력
- 사용자 자유 입력 (한 문단~여러 문단)

3) 출력 옵션 선택
- 실행 프롬프트 (필수)
- Markdown 문서 (옵션)

4) 생성 버튼
- 생성된 프롬프트 복사/다운로드

## 6. Functional Requirements
FR-1. Project Core 문서 로드
FR-2. Sub-Agent 문서 목록/선택
FR-3. 자유 입력 받기
FR-4. 실행 프롬프트 생성
FR-5. (옵션) 문서형 출력 생성
FR-6. (옵션) 응답 저장 및 재사용 (Responses API store 기반)

## 7. Quality Requirements
- 빠른 반응 (체감 2~5초 목표)
- 맥락 누락 최소화 (필요 문서만 정확히 결합)
- 보안: API Key는 클라이언트에 노출 금지 (서버 라우트에서 호출)

## 8. Success Metrics (개인 OS)
- 프롬프트 작성 시간 80% 이상 감소
- 수정/확장 시 “망가짐” 체감 감소
- 반복 작업에서 프롬프트 재사용률 증가
